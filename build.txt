03 | SECTION 1: HERO (Scroll-Triggered Video)

All behavior, frame generation, canvas animation, progressive batch
loading, and responsive rules remain IDENTICAL to the original template
spec. Only the visual content and text change.

⚠ HERO VIDEO SETUP — REQUIRED (do this first or hero section will be blank):

Step 1: Download a free dark cybersecurity video from Pexels.com
        Search term: 'cyber network dark' or 'hacker terminal'
        Download as MP4 and rename to: hero-video.mp4
        Place it in: /public/hero-video.mp4

Step 2: Generate frames from the video using this exact ffmpeg command:
        mkdir -p public/frames
        ffmpeg -i public/hero-video.mp4 \
        -vf "fps=24,scale=1920:1080" \
        -c:v libwebp -quality 85 \
        -compression_level 6 \
        -start_number 1 \
        public/frames/frame_%04d.webp

Step 3: Generate the poster (first frame) using this command:
        ffmpeg -i public/hero-video.mp4 -vframes 1 public/video-poster.webp

Step 4: In the Hero component (src/components/Hero.jsx or equivalent),
        make sure the canvas reads frames from: /frames/frame_%04d.webp
        The canvas must be set to: width="1920" height="1080"
        position: absolute; top: 0; left: 0; width: 100%; height: 100%;
        object-fit: cover;

Step 5: ScrollTrigger must be linked to the canvas frame counter.
        As the user scrolls down, the frame index increments.
        Use GSAP ScrollTrigger scrub: true on the frame counter variable.
        Frame range: frame_0001.webp → last frame (depends on video length).

⚠ Without completing Steps 1–3, the hero section will render as a black
empty screen. The canvas REQUIRES the /public/frames/ folder to exist
with at least frame_0001.webp before the page loads.
